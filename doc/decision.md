## 1. 프로젝트 범위

### 결정: Phase 1 = 수집 시스템만
**날짜:** 2026-04-24
**이유:**
- 영상까지 한꺼번에 만들면 너무 큼 (2-4주 작업)
- 작은 거 작동시키고 → 거기에 추가하는 방식
- 수집이 작동해야 영상 입력 데이터 있음

**Phase 1 포함:**
- 뉴스 수집
- 카테고리 분류
- 중복 감지
- WebApp UI
- SBS 자막 분석

**Phase 2로 미룸:**
- 영상 자동 생성
- 본인 음성 (TTS)
- 자동 YouTube 업로드
- 광고 삽입

## 2. SBS/JTBC 영상

### 결정: 영상 클립 사용 안 함
**이유:**
- Content ID 자동 감지 (3-4초 잘라도 걸림)
- 공개 채널이면 차단/경고 위험
- 합법적으로 회피 불가능

**대신:**
- 자막만 추출 → 학습/참고용
- 본인 영상에는 링크/임베드만
- 사실 정보만 차용 → 본인 표현으로 재작성

---

## 3. AI 이미지 생성

### 결정: 사용 안 함
**이유:**
- 본인 방침: "가짜 느낌" 싫음
- 실존 정치인 닮은 이미지 = 윤리적 회색
- 딥페이크 규제 강화 추세

**대신:**
- Pexels/Pixabay 무료 사진/영상
- 한국 매체 사진 + 출처 표기
- 본인 그래픽/인포그래픽

---

## 4. 저작권 정책

### 결정: 출처 표기 + 회색지대 운영
**이유:**
- 100% 합법만 하면 비주얼 너무 제한적
- 출처 표기하면 실질적 위험 매우 낮음
- 한인 채널 99%가 이렇게 운영 중

**원칙:**
- 모든 사진 출처 자동 표기
- 사진 5초 이내 노출
- Getty/AP 워터마크 사진 X
- 영상 클립 X
- 음악 저작권 X
- 권리자 요청 시 즉시 삭제

---

## 5. 채널 공개 범위

### 결정: 비공개 또는 unlisted
**이유:**
- 공개하면 저작권 위험 큼
- 엄마 + 친한 사람 정도면 충분
- 한인 사회 입소문 위험

**나중에 변경 가능:**
- 콘텐츠 100% 합법으로 전환 시
- 또는 채널 정체성 확립 후
